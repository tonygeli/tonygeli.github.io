OO目标

高內聚，低耦合

OO设计基本原则

  • 单一职责：一个类只做一件事
  • 依赖倒置：设计依赖于抽象而不是去实现
  • 最少知识：对象应当尽可能少的去了解其他对象
  • 组合优先：优先使用类的组合而不是继承

构造型设计模式

  • 单例模式
  • 抽象工厂模式
  • 构造模式
  • 原型模式
  • 工厂方法模式

单例(Singleton)

  • 全局只有一个类的对象
  • 私有化构造方法
  • 通过静态方法创建并保持对象
  • 注意是梨花方法需要线程安全

适用场景：

  • 类的对象全局唯一
  • 节约频繁创建对象的资源开销
  • 单例类必须是无状态的

抽象工厂(Abstract Factory)

  • 使用和实现分离
  • 调用方只依赖于业务逻辑接口，与具体实现无关

适用场景:

  • 剥离调用者和实现的耦合，应对变化
  • 调用者、实现代码可以分别独立测试
  • 典型案例:JDBC API & Driver

构造者(Builder)

  • 用于创建一个复杂对象
  • 一个对象需要依赖于大量外部对象或者其他设置参数，通过构造方法构建太复杂

适用场景:

  • 对象的创建非常复杂，总是需要很多配置和预先创建其他对象
  • 抽象的构造者还能实现相同的步骤创建出不同的对象

原型(Prototype)

  • 新的对象由已有对象复制而来
  • 一般出现在值对象中，原有值对象不可变，改变值总是复制新的对象

适用场景:

  • 由原有对象创建新对象
  • 原对象不可变，必须复制对象后再改写
  • 典型实例：JavaScript
