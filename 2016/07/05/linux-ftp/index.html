FTP虚拟用户

  1. 生成虚拟用户口令库文件

    创建login.txt
    vim login.txt
    guest #user
    guest #psw

  2. 配置生成VSFTPD的认证文件

    使用db_load命令生成口令库文件
    db_load -T -t hash -f login.txt /etc/vsftpd/vsftpd_login.db
    修改文件权限
    chmod 600 /etc/vsftpd/vsftpd_login.db
    注：无db_load命令，使用yum install db4*
    编辑PAM配置文件
    vim /etc/pam.d/vsftpd
    将所有内容注释后，添加如下两行：
    auth required /lib/security/pam_userdb.so db=/etc/vsftpd/vsftpd_login
    account required /lib/security/pam_userdb.so db=/etc/vsftpd/vsftpd_login

  3. 建立虚拟用户的访问权限

    建立虚拟用户virtual并设置目录及权限
    useradd -d /home/ftp virtual
    chmod 700 /home/ftp

  4. 建立配置文件

    VSFTPD主配置文件中修改：
    local_enable=YES
    guest_enable=YES //启动虚拟用户
    guest_username=virtual //映射到本地virtual用户
    service vsftpd restart

虚拟FTP服务

  1. 创建虚拟FTP服务器的根目录

    创建/var/newftp/newpub 并将目录所有者设为root，权限为755
    mkdir -p /var/newftp/newpub
    chmod 755 /var/newftp -R

  2. 添加虚拟FTP服务器的匿名用户帐号

    useradd -d /var/newftp - M newftp
    原来FTP服务匿名用户帐号为ftp
    虚拟FTP服务匿名用户帐号为newftp
    复制原来的vsftpd.conf 作为虚拟FTP服务器的配置文件，并修改相关参数
    cp /etc/vsftpd/vsftpd.conf /etc/vsftpd/vsftpd2.conf
    listen=YES
    listen_address=192.168.1.13
    ftp_username=newftp
    在原来的vsftpd.conf中添加
    listen_address=192.168.1.11
